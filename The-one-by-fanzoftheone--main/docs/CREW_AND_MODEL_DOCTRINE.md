# CREW AND MODEL DOCTRINE — THE ONE

THE ONE ecosystem separates **agents (crew)** from **models (engines)**.

This prevents dependency on any single AI provider and allows creators to own their systems.

---

## Core Principle

THE ONE is the command system.

Models are replaceable engines.

Agents are the crew.

Marketplace is the earning layer.

Training is the ownership layer.

---

## System Roles

### THE ONE Runtime
The command and orchestration layer.

Responsible for:

- agent coordination
- tool access
- memory
- workflow execution
- routing model calls

---

### The Crew (Agents)

Agents represent working roles inside the system.

Examples:

- planner
- worker
- auditor
- coordinator
- builder
- researcher

Agents do not belong to any model provider.

Agents operate through the runtime and can use any model lane available.

---

### Model Lanes

Models are treated as **engines**.

Examples:

- local models
- hosted APIs
- creator-trained models

Possible providers include:

- local inference
- community models
- external APIs

These engines are interchangeable.

---

## Creator Ownership Layer

Creators can:

- train their own model lanes
- publish model lanes
- register tools
- sell outputs
- build products from agent workflows

Ownership remains with the creator.

---

## Marketplace Layer

Outputs created through THE ONE can become products.

Examples:

- agent workflows
- trained model lanes
- automation modules
- digital tools
- service applications

Creators can list these products in the marketplace.

---

## Architectural Model

THE ONE runtime

↓  

Agent Crew

↓  

Model Gateway

↓  

Model Lanes

↓  

Products and Marketplace

---

## Philosophy

Agents are the crew.

Models are tools.

The system belongs to its creators.

THE ONE exists to give builders freedom to create, train, publish, and earn without dependency on any single AI provider.
