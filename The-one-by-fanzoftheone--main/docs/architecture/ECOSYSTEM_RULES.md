# ECOSYSTEM RULES — THE ONE

These rules protect the architecture of THE ONE ecosystem.

---

## Core Separation Rules

Apps must not contain modules.

Modules must not contain UI pages.

Agents must not live inside apps.

Runtime must remain separate from agents.

Warehouse must only store memory, logs, and state.

Workers must only execute tasks.

Tools must provide capabilities but not contain workflows.

---

## Folder Responsibilities

apps/
- product experiences
- UI flows
- business usage

modules/
- reusable logic
- shared capabilities
- system building blocks

agents/
- autonomous behavior
- decision logic
- orchestration helpers

runtime/
- execution loops
- agent coordination
- scheduling

warehouse/
- long term memory
- state storage
- logs

tools/
- capabilities
- integrations
- system utilities

labs/
- experiments
- prototypes
- new ideas

forge/
- app builder
- module generator
- system creator

marketplace/
- installable apps
- installable modules
- installable agents

---

## Promotion Lifecycle

Nothing skips stages.

labs → modules → apps → marketplace

---

## Build Order

All new capabilities follow this order:

1 backend route  
2 frontend connector  
3 ecosystem UI  
4 app usage  
5 module promotion  
6 marketplace listing  

---

## Forbidden Architecture

Do NOT create:

apps inside modules

agents inside apps

modules inside frontend UI

runtime inside apps

---

## Purpose

These rules keep THE ONE scalable, modular, and monetizable.
