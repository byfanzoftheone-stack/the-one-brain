# THE ONE — No Drift Rules

## Rule 1 — Every Feature Must Have a Home

Nothing floats outside the system.

Every idea must belong to one of:

labs
modules
apps
marketplace
warehouse
companion
integrations


## Rule 2 — No Idea Goes Straight To App

Ideas must move through the pipeline.

idea
↓
lab experiment
↓
validated pattern
↓
module
↓
app
↓
marketplace


## Rule 3 — Labs Are Experimental

Labs are where ideas are tested.

Labs may contain:
- prototypes
- experiments
- unfinished logic
- AI behavior testing

Labs are **not products**.


## Rule 4 — Modules Are Stable Capabilities

A module must:

- solve a repeatable problem
- have defined inputs/outputs
- have a module.json manifest
- be installable by Builder


## Rule 5 — Apps Are Monetized Systems

Apps are full products built from modules.

Examples:

construction-ops
poolconnect
cleaning-ops
playbetter
not-alone


## Rule 6 — Marketplace Distributes Products

Marketplace distributes:

modules
apps
agents
templates


## Rule 7 — Companion Stays A Growth Partner

Companion is:

friend
mentor
thinking partner
accountability mirror

Companion is NOT:

romance AI
therapist
generic assistant


## Rule 8 — User Remains Orchestrator

The user decides:

what gets built
what gets promoted
what becomes a module
what becomes an app

No hidden system replaces the user.


## Rule 9 — Repo Is The Source Of Truth

Architecture must live in the repository.

Conversation memory cannot override repository truth.


## Rule 10 — Pipeline Before Code

Before building anything ask:

Is it a lab?
Is it a module?
Is it an app?

Only then write code.
