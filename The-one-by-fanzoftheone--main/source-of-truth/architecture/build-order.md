# THE ONE — Locked Build Order

## Rule
Do not build everything at once.
Do not drift into side systems before the active lane is stable.

## Correct implementation order

1. Stabilize shell
- layout
- navigation
- mobile control frame
- hub
- builder
- agents
- marketplace
- labs
- companion
- settings

2. Stabilize backend core
- health route
- module loader
- companion route
- deployment pipeline

3. Build revenue lane first
- module registry
- module install/export
- builder generator
- marketplace module feed
- app generation flow

4. Add warehouse memory layer
- prompt vault
- receipts
- code backpack
- conversation history
- archives

5. Add agent runtime
- teacher compare logic
- student runtime
- skip execution loop
- backpack write flow

6. Add companion intelligence
- reflection engine
- memory logic
- challenge / reframe loop
- pattern detection
- planning support

7. Add labs runtime
- flow labs
- agent labs
- prototype builder
- little pro experiments

8. Add marketplace publishing
- module publishing
- app publishing
- pricing
- licensing

## Rule of priority

Money lane first:
Builder + Modules + Marketplace + Apps

Growth lane stays first-class:
Companion + Memory + Reflection

Do not bury Companion.
Do not let Companion replace the revenue lane.
