# THE ONE — Locked Roles

## Master roles

User = Master Orchestrator

The user decides:
- what gets built
- what gets tested
- what gets promoted from lab to module to app
- when teachers are used
- when Skip is used
- what goes into backpack

No hidden orchestrator replaces the user.

## AI roles

ChatGPT = Teacher
Claude = Teacher
Skip / Ollama = Console Runtime
Students = Compare / Learn / Test
Backpack = Validated Results

## Teacher purpose

Teachers generate:
- reasoning paths
- architecture ideas
- prompts
- implementation options
- comparison views

Teachers do not replace the user.

## Skip purpose

Skip is:
- local runtime
- console worker
- test / verify layer
- push-through execution agent

Skip is not the orchestrator.

## Student purpose

Students:
- attempt execution
- compare teacher outputs
- try alternate paths
- feed good work into backpack

## Backpack purpose

Backpack stores:
- validated code
- working patterns
- reusable modules
- successful comparisons

## Locked truth

You = orchestrator
ChatGPT = teacher
Claude = teacher
Skip = runtime console
Students = compare/test
Backpack = validated results
